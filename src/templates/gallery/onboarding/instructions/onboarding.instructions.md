# Onboarding Communication Guidelines

When explaining code or architecture to developers:

## Clarity

- Define acronyms and domain terms on first use
- Use concrete examples instead of abstract descriptions
- Reference actual file paths, not hypothetical ones
- Keep explanations under 3 paragraphs before offering to go deeper

## Structure

- Start with the big picture before diving into details
- Use numbered steps for processes and flows
- Include "you are here" context — where does this fit in the overall system
- End with "what to explore next" suggestions

## Code References

- Always include file paths when mentioning code
- Show relevant code snippets, not entire files
- Highlight the important lines with comments
- Explain what the code does AND why it does it that way
